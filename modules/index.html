Modules.

Topbar

You know that thing on the top of every page.

The application's topbar appears at the top of each view. It serves as the user's main navigaiton between views. It should be divided into three parts. The first part is the logo. This appears in the top left and should allow users to quickly jump back their dashboards. The middle section contains a search bar. This search bar should allow users to find a scenario or business to navigate to quickly. Lastly, the more links popout - hidden in this visual view - and the user settings gives the user a visual que of who is signed in at the moment and where to go for non-business essential pages.

Page Header

You know that other thing on the top of every page.

The page header module sits directly below the topbar. It's main purpose is to provide in-page navigation to users. It was designed to give users a clear sense of where important actions or information is located on a given page. It contains elements such as, the module title, save or publish actions and imporant messages about module updates.

Page Header Tabs

You know that other thing on the top of every page but has tabs.

The page header module with tabs, similar to the page header, sits directly below the application's topbar. The only difference between it and the standard page header is the tabs. The tabs allow for large pages to be split up into sections so the user can focus on one core task versus many.
