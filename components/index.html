Alerts

Oh no! Did you break something?

We highly doubt it, but this would be a good time to let the user know something went wrong. Alerts, should be used when the user needs instant feedback on an action they preformed. They are helpful when a user deletes a module or when they have successfully saved or created something new. Below you will find three types of alerts and how to best use them in the interface.

Badges

Hey you! Yeah you. Look over here.

Now that we have your attention, let's talk about badges and when to use them. Badges should be used to label new features, items that might be broken, or notify a user about the number of items they have selected. They should call just enough attention so a user notices them, but they do not need to be the dominate element on the page or item.

Buttons

Click me! Go ahead do it.

Let's talk about buttons. These elements should be used for actions that the user can take. The primary button is labeled alpha and is the domination application color. It should be used in most cases when users are prompted to perform a submission that can be undone. On the other hand, the charlie button should be used for actions that cannot be undone or are final. The button styles can be applied to various elements on the page. As long as it is suppose to act like a button, the styles are OK to use.

Sample Code

  <!-- Standard Button -->
  <button class="button--alpha">Button Text</button>

  <!-- Anchor Button -->
  <a href="#" role="button" class="button--alpha">Button Text</a>

  <!-- Input Type Submit Button -->
  <input type="submit" name="submitButton" value="Button Text" class="button--disabled" disabled>

HTML Guidelines

This table gives you a quick overview of the CSS classes that can be applied to create a button.

Class Name Usage
.button--[type] This should be applied to the element you are styling to look like a button. The class name should change based on the type of button being used. 
<button class="button--alpha">Button Text</button>
.button--alpha Standard button and should be used in almost all cases.
.button--bravo Button used for in-page actions such as copying text to a user's clipboard.
.button--charlie Only use this button when an action is final and cannot be undone.
.button--disabled This style of button should be used to notify a user they cannot perform an action because of missing data or there are no changes to be saved.
role="button" If you style a link to look like a button please add this attribute to maintain a level of semantic markup.

Chips

Crunch. Crunch. Crunch.

A chip is made up of two things: an image and some content. They are fairly universal and can be used in many situations, but they must contain these two elements. The image sits on the left side of the chip and has a 100% border radius which gives it the circle look. The text sits on the right side of the chip. The text may be a description, combination of a name and date or title and subtitle. On smaller screens, the chip image is hidden to make room for the text.

Links

What a beautiful day to hit the links.

This component should be used for navigational purposes or appending items to a list. If you want to move a user from one page to the next or load an overlay, then a link is the right way to go. Avoid using links when asking users to submit final actions. They can be used next to a submit action if want to provide users a way to cancel or close an item, however.

This table gives you a quick overview of the CSS classes that can be applied to create tabs.

Class Name Usage

<div class"tabs"></div>

<ul class"tab__list"></ul>

Toggle

Now you see me. Now you don't.

Do you need to present a lot of content or items, but don't want them to take up a lot of space? A toggle might be a good way to display this content then. Toggles are meant to allow users to hide and show content when they need it.

.is-active On click, this class should be added to the toggle trigger and toggle body.
.toggle--border Add this modifier to the toggle parent if you would like to add a border to the bottom of the toggle element. Good for lists of toggle elements. 
<div class="toggle toggle--border"></div>
.toggle--card Add this modifier to the toggle parent if you would like to style the toggle element like a card. 
<div class="toggle toggle--card"></div>

Tooltips

Need a little help or guidance?

If you want to provide some information about a feature or form element, a tooltip is a great idea. This component should be used to provide just enough information to help users understand a particular part of the interface. Keep the content inside of the tooltip to a minimum.
